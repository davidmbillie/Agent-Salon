# Design notes

## Guiding idea

The curator owns identity continuity. Models may interpret, converse, and make
suggestions, but they do not silently decide what becomes canonical memory.

## Open questions

- Should the first interface be a terminal conversation, a small local web app,
  or both?
- Should both personas see the complete transcript or receive only relayed turns?
- How should disagreements over proposed memories be displayed?
- Which existing instructions and conversations best capture each persona?
- Should coding sessions and whimsical conversations share memory or remain
  separate contexts?

## Decisions

- 2026-07-17: Begin with provider-neutral folders before selecting Python or
  TypeScript.
- 2026-07-17: Memory promotion always requires curator approval.
